Secara umum, proses kerja di AI Department terdiri dari:

### 1. Request Intake
Semua pekerjaan dimulai dari kebutuhan bisnis. Form request untuk departemen AI dapat dilihat di <Link to="https://forms.gle/vc8sdASUeFsaPjT6A">AI Request Form</Link>.

### 2. Research
|        |   |
|------------|------|
| **Tujuan**     | Validasi feasibility teknis |
| **Requester**  | <Badge variant="default">C-Level</Badge> <Badge variant="default">Bisdev</Badge> |
| **Durasi**     | 2–7 hari |
| **Output**     | Report, Model |

<details>
<summary>Detail</summary>
:::info

Riset bertujuan menjawab apakah suatu ide memungkinkan secara teknis, bukan membangun solusi final.

:::

Contoh pertanyaan:
- "Bisakah object detection digunakan untuk mengukur okupansi toko?"
- "Berapa batas pengukuran depth camera dari ketinggian tertentu?"

Gunakan template berikut untuk report:  
<Link to="https://docs.google.com/document/d/1sLtLpA9QeOQZYJacwBMw-u1pnVL8ODSr5XTYZwGxpwc/edit?usp=sharing">
Research Report Template
</Link>

Panduan lengkap tersedia di <Link to="../research">Research Guide</Link>

</details>

### 3. Prototyping
|        |   |
|------------|------|
| **Tujuan**     | Memvalidasi solusi ke user atau calon clien |
| **Requester**  | <Badge variant="default">C-Level</Badge> <Badge variant="default">Bisdev</Badge> |
| **Durasi**     | 5–10 hari |
| **Output**     | Prototype yang dapat didemonstrasikan |

<details>
<summary>Detail</summary>

:::warning

Prototyping berfokus pada validasi, bukan kesempurnaan.

:::

Karakteristik:
- Fitur minimal
- Fokus pada use case utama
- Cepat dibuat dan diuji

Terdapat dua jenis:
- PoC (internal validation)
- Paid PoC (digunakan oleh client dalam periode tertentu)

Pendekatan pengembangan berbeda untuk masing-masing jenis.

Panduan lengkap tersedia di <Link to="../prototyping">Prototyping</Link>

</details>


### 3. Development
|        |   |
|------------|------|
| **Tujuan**     | Membangun solusi production-ready. |
| **Requester**  | <Badge variant="default">Project Manager</Badge> <Badge variant="default">Product Manager</Badge> |
| **Durasi**     | - |
| **Scope**     | <Badge variant="default">Pengembangan Produk</Badge> <Badge variant="default">Project</Badge> <Badge variant="default">Testing</Badge>|
| **Output**     | Sistem siap deploy |

<details>
<summary>Detail</summary>

Pada tahap ini, engineer bekerja berdasarkan requirement yang sudah tervalidasi. Engineer bertanggung jawab kepada Project/Product Manager.

Flow dan SOP dapat dilihat di  
<Link to="../development">Development</Link>

</details>

### 4. Deployment
|        |   |
|------------|------|
| **Tujuan**     | Menyediakan solusi agar dapat digunakan di environment target. |
| **Scope**     | <Badge variant="default">Cloud Deployment</Badge> <Badge variant="default">Edge Deployment</Badge>|
| **Output**     | Servis atau aplikasi running |

<details>
<summary>Detail</summary>
:::info

Deployment mencakup proses packaging, konfigurasi, dan integrasi dengan sistem lain.

:::

Panduan lengkap mengenai deployment tersedia di <Link to="../deployment">Deployment</Link>

</details>

### 5. Monitoring & Iteration
|        |   |
|------------|------|
| **Tujuan**     |Menjaga performa dan melakukan *continuous improvement*.|
| **Scope**     | <Badge variant="default">Documentation</Badge> <Badge variant="default">Bug Fixing</Badge><Badge variant="default">Model Monitoring</Badge> <Badge variant="default">App Monitoring</Badge>|
| **Output**     | Sistem running dengan performa stabil |
**Tujuan**  

<details>
<summary>Detail</summary>

Tahap ini berlangsung setelah deployment dan bersifat berkelanjutan.

Mencakup:
- Monitoring performa model
- Dokumentasi project
- Perbaikan bug
- Iterasi model atau sistem

Panduan lengkap tersedia di  
<Link to="../monitoring">Monitoring</Link>

</details>
