## Ringkasan

Halaman ini menjelaskan standar praktis pengembangan model Computer Vision. Fokus utama: **reproducibility** dan **dokumentasi** — bagaimana agar developer lain bisa mengulang eksperimen, memverifikasi hasil, dan melanjutkan pekerjaan tanpa menebak‑nebak konfigurasi.

Scope: object detection, classification, segmentation. Untuk anotasi, kita menggunakan **Roboflow** sebagai tooling standar (lihat bagian Anotasi).

## Prinsip Utama

- Dokumentasikan setiap eksperimen: dataset version, commit hash, config file, seed, dan lingkungan (packages).
- Simpan artefak minimal: trained model, evaluation report, dataset manifest, training logs, model card.
- Buat eksperimen bisa diulang: satu perintah untuk reproduce (script/runbook) dan environment pinning.

## Data Collection & Annotation (Roboflow)

- Gunakan Roboflow untuk manajemen dataset dan anotasi. Simpan workspace/project ID dan versi export di dokumentasi eksperimen.
- Aturan labeling singkat (untuk developer yang membuat spec annotator):
  - Buat `label taxonomy` yang jelas dan versioned (mis. labels_v1.yaml) terbuat auto di roboflow.
  - Sertakan contoh positif/negatif untuk tiap kelas.
  - Document ambiguitas dan edge cases.

Catat: project ID, version number, export format, dan hash file hasil export.

## Dataset Versioning

- Simpan manifest sederhana (JSONL/JSON) yang berisi: file path, split (train/val/test), source, license, annotation_version.
- Versioning: pakai semantic-like tag `dataset-name/YYYYMMDD-vN` atau commit hash dari tempat penyimpanan (contoh: s3://bucket/datasets/ppe/20260501-v1).

Contoh minimal manifest (manifest.json):

```json
{
  "name": "ppe-dataset",
  "version": "2026-05-01-v1",
  "source": "roboflow:my-project:5",
  "license": "internal",
  "files": 12345
}
```

## Experiment Tracking & Logs

- Gunakan satu alat tracking (W&B atau MLflow) untuk menyimpan metrics, artifacts, dan run config. Link ke run harus ada di PR/issue yang berhubungan.
- Jika tidak dapat online, simpan folder `runs/<id>/` yang berisi metrics.json, checkpoints, and run_metadata.txt.

## Evaluation & Model Selection

- Tentukan primary metric tergantung task (contoh: detection -> mAP@0.5, classification -> F1). Catat evaluasi pada dataset yang versioned.
- Simpan evaluation report (CSV/JSON) yang memuat per-class metrics, confusion matrix, dan sample failure cases (gambar + prediction + gt).

## Model Card & Artefak

Untuk setiap model yang diusulkan ke deployment, buat model card singkat berisi:

- model_name, version
- task, architecture
- dataset_version
- metrics (primary + secondary)
- intended use & limitations

Contoh snippet model_card.yaml:

```yaml
model_name: ppe-detector
version: v0.1.0
task: object-detection
dataset_version: 2026-05-01-v1
metrics:
  mAP@0.5: 0.78
  latency_ms: 35
limitations:
  - Belum dites untuk scene malam hari
```

Simpan model card di `models/<model_name>/model_card.yaml` dan sertakan link.

## Deployment & Monitoring (ringkas)

- Pastikan model yang dideploy memiliki model card dan smoke test yang dapat dijalankan oleh orang lain.
- Packaging: sediakan script untuk export model ke format yang dipakai production (TorchScript/ONNX) dan dokumentasikan perbedaan numerik jika ada.
- Monitoring: definisikan metrics yang dipantau (latency, error-rate, prediction-distribution) dan tempat menyimpan logs/telemetry.

## PR Checklist (Model PR)

Tambahkan checklist ini ke deskripsi PR yang mengusulkan model baru atau update:

- [ ] Dataset manifest & Roboflow project/version tercantum
- [ ] Run metadata (requirements, seed, command) tersedia di artifact atau run_metadata.txt
- [ ] Experiment tracking link disertakan (W&B/MLflow) atau folder report
- [ ] Model card tersedia di
- [ ] Evaluation report dan sample failure cases dilampirkan

## Tips Praktis

- Prioritaskan reproducibility: lebih baik ada run yang bisa diulang walau performa sedikit lebih rendah.
- Dokumentasi > micro-optimizations saat eksperimen awal. Fokuskan effort menulis runbook singkat untuk tiap eksperimen.
- Jika menggunakan Roboflow: snapshot versi export setiap kali memperbarui anotasi.
