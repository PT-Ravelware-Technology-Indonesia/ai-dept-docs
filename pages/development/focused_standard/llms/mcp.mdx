title: MCP (Model Context Protocol)
---

**MCP (Model Context Protocol)** itu gampangnya adalah **"remote tools buat LLM"**.

Ada dua sisi:

| Peran         | Artinya                                                                 |
|---------------|-------------------------------------------------------------------------|
| **Consumer**  | Kamu pake MCP buat ngasih tools ke LLM kita secara remote. LLM bisa akses database, file system, API — tinggal connect. |
| **Producer**  | Kamu bikin MCP server biar aplikasi kita **AI Agent Ready** — kayak API khusus buat LLM. Aplikasi kita bisa dipake langsung sama agent AI tanpa integrasi manual. |

### Kapan pake MCP?

- **Consumer**: Kalo LLM agent kita butuh akses data real-time atau eksekusi actions di luar chat
- **Producer**: Kalo kita punya aplikasi/service dan mau bisa dipanggil langsung sama AI agent — seperti: "Bikin user baru", "Cari data transaksi", "Generate report" — tanpa harus nulis integrasi manual

## Agent Pake MCP

Di <Link to="/development/focused_standard/llms/ai_agents">halaman AI Agent</Link> kita pake tools dengan manggil fungsi Python langsung (local function). Dengan MCP, tools itu nggak musti local — bisa **remote**.

Jadi kalo agent sebelumnya panggil `web_search()` sebagai function langsung, dengan MCP agent bisa panggil tool dari server remote — kayak `query_database()`, `post_to_slack()`, atau tool kustom dari aplikasi kita.

### Contoh: Agent Pake MCP Tools

```python title="agent_with_mcp.py"
import asyncio
from openai import OpenAI
from pydantic import BaseModel
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client


# ─── Agent Decision Schema ──────────────────────────────────────

class Decision(BaseModel):
    action: str
    tool_name: str | None = None
    tool_args: dict | None = None
    response: str | None = None


# ─── MCP Client ─────────────────────────────────────────────────

class MCPToolClient:
    """Wrapper buat konek ke MCP server dan dapetin tool list."""

    def __init__(self, server_command: str, server_args: list[str]):
        self.params = StdioServerParameters(
            command=server_command,
            args=server_args,
        )
        self.tools: list[dict] = []

    async def connect(self):
        self._read, self._write = await stdio_client(self.params).__aenter__()
        self.session = await ClientSession(self._read, self._write).__aenter__()
        await self.session.initialize()
        tools = await self.session.list_tools()
        self.tools = [
            {"name": t.name, "description": t.description}
            for t in tools.tools
        ]
        return self.tools

    async def call(self, name: str, arguments: dict) -> str:
        result = await self.session.call_tool(name, arguments)
        return result.content[0].text if result.content else "No result"

    async def disconnect(self):
        await self.session.__aexit__(None, None, None)
        await self._write.close()


# ─── Agent Loop ─────────────────────────────────────────────────

SYSTEM_PROMPT = """Kamu adalah asisten dengan akses ke MCP tools.
Pilih tool yang sesuai atau langsung jawab kalo udah cukup."""


async def run_agent(question: str, mcp_tools: list[dict]):
    client = OpenAI(base_url="https://openrouter.ai/api/v1",
                    api_key="...")
    messages = [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": question},
    ]
    tools_desc = "\n".join(
        f"- {t['name']}: {t['description']}" for t in mcp_tools
    )

    for step in range(5):
        resp = client.beta.chat.completions.parse(
            model="google/gemini-3-flash-preview",
            messages=messages + [
                {"role": "user", "content": f"Available tools:\n{tools_desc}\nApa action kita?"}
            ],
            response_format=Decision,
        )
        decision = resp.choices[0].message.parsed

        if decision.action == "answer":
            return decision.response

        print(f"[Step {step}] Calling MCP tool: {decision.tool_name}({decision.tool_args})")
        result = await mcp_client.call(decision.tool_name, decision.tool_args or {})
        print(f"  Result: {result[:100]}...")
        messages.append({"role": "assistant", "content": f"Tool result: {result}"})

    return "Max steps reached"


# ─── Run ────────────────────────────────────────────────────────

async def main():
    server = MCPToolClient(
        server_command="uvx",
        server_args=["mcp-server-sqlite", "--db", "data.db"],
    )
    tools = await server.connect()
    print(f"MCP tools ready: {[t['name'] for t in tools]}")

    answer = await run_agent("Berapa total user di database?", tools)
    print(f"\nAnswer: {answer}")

    await server.disconnect()


asyncio.run(main())
```

**Alurnya:**

1. Agent connect ke MCP server — dapet daftar tools yang tersedia
2. Di tiap step, agent decide: **pake tool** atau **jawab langsung**
3. Kalo pake tool, agent specify `tool_name` + `tool_args`, hasilnya dikembalikan ke agent
4. Agent process hasil tool, lalu decide lagi — sampai cukup buat jawab

Ini persis sama seperti ReAct loop di halaman AI Agent, bedanya tools-nya di-remote via MCP.

## Cara Bikin MCP Server (Producer)

Kalo kita punya aplikasi dan mau **AI Agent Ready** — tinggal bungkus jadi MCP server. Anggep aja kayak **bikin API, tapi khusus buat LLM**.

### 1. Init Project

```bash
mkdir mcp-server && cd mcp-server
uv init
uv add "mcp[cli]"
```

### 2. Bikin Server — Calculator

```python title="server.py"
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent


app = Server("calculator")


@app.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="add",
            description="Add two numbers together",
            inputSchema={
                "type": "object",
                "properties": {
                    "a": {"type": "number"},
                    "b": {"type": "number"},
                },
                "required": ["a", "b"],
            },
        ),
    ]


@app.call_tool()
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    if name == "add":
        result = arguments["a"] + arguments["b"]
        return [TextContent(type="text", text=str(result))]
    raise ValueError(f"Unknown tool: {name}")


async def main():
    async with stdio_server() as (read, write):
        await app.run(read, write)


if __name__ == "__main__":
    import asyncio
    asyncio.run(main())
```

```bash
mcp run server.py
```

### 3. Bikin Server Realistis — Notes API

```python title="notes_server.py"
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent, Resource


app = Server("notes")
notes: dict[str, str] = {}


@app.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="create_note",
            description="Create a new note",
            inputSchema={
                "type": "object",
                "properties": {
                    "title": {"type": "string"},
                    "content": {"type": "string"},
                },
                "required": ["title", "content"],
            },
        ),
        Tool(
            name="get_note",
            description="Get a note by title",
            inputSchema={
                "type": "object",
                "properties": {"title": {"type": "string"}},
                "required": ["title"],
            },
        ),
    ]


@app.call_tool()
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    match name:
        case "create_note":
            notes[arguments["title"]] = arguments["content"]
            return [TextContent(type="text", text=f"Created: {arguments['title']}")]
        case "get_note":
            content = notes.get(arguments["title"])
            if content is None:
                raise ValueError("Note not found")
            return [TextContent(type="text", text=content)]
        case _:
            raise ValueError(f"Unknown tool: {name}")


@app.list_resources()
async def list_resources() -> list[Resource]:
    return [
        Resource(
            uri=f"note:///{title}", name=title,
            mimeType="text/plain",
        )
        for title in notes
    ]


@app.read_resource()
async def read_resource(uri: str) -> str:
    title = uri.replace("note:///", "")
    return notes.get(title) or ""


async def main():
    async with stdio_server() as (read, write):
        await app.run(read, write)


if __name__ == "__main__":
    import asyncio
    asyncio.run(main())
```

### 4. Transport Options

| Transport | Kapan Dipake                        | Cara Run                              |
|-----------|-------------------------------------|----------------------------------------|
| **stdio** | Lokal — cocok buat development      | `mcp run server.py`                    |
| **SSE**   | Remote — biar bisa diakses agent lain | `mcp run --transport sse server.py`   |

**SSE** — kalo server kita perlu diakses dari mesin lain:

```python
from mcp.server import Server
from mcp.server.sse import SseServerTransport
from starlette.applications import Starlette
from starlette.routing import Route


app = Server("my-app")
sse = SseServerTransport("/messages")


async def handle_sse(request):
    async with sse.connect_sse(
        request.scope, request.receive, request._send
    ) as (read, write):
        await app.run(read, write)


async def handle_messages(request):
    await sse.handle_post_message(
        request.scope, request.receive, request._send
    )


starlette_app = Starlette(routes=[
    Route("/sse", endpoint=handle_sse),
    Route("/messages", endpoint=handle_messages, methods=["POST"]),
])
```

### 5. Best Practices

1. **Deskripsi tool yang jelas** — LLM rely on deskripsi buat milih tool, jadi tulis yang gamblang
2. **Input schema yang strict** — define required/optional fields biar LLM nggak bingung
3. **Error messages yang informatif** — kalo error, return pesan yang bisa dipahami LLM biar dia bisa coba lagi
4. **Security** — MCP server punya akses penuh ke resource-nya. Jangan expose ke publik tanpa auth kalo pake transport SSE
5. **Resources ≠ Tools** — Resources buat **data** (bisa dibaca langsung), Tools buat **actions** (yang ngejalanin sesuatu)
