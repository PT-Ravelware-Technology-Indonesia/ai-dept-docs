title: Observability Guide

Dokumen ini menjelaskan standar observability general untuk seluruh sistem di AI Department. 
Observability mencakup logging, metrics, dan tracing yang memungkinkan sistem dapat dipantau, dianalisis, dan di-debug dengan efektif.
masing-masing fokus seperti computer vision dan LLMs memiliki standard observability khusus yang dapat dilihat di halaman observability masing-masing fokus.

## Tujuan

Observability bertujuan untuk:

- Memahami kondisi sistem secara real-time  
- Mempermudah debugging dan incident investigation  
- Mendeteksi masalah lebih awal  
- Menjaga reliability dan performance sistem  

---

## Prinsip Utama

- **Everything important is observable**  
- **Structured over unstructured**  
- **Context is critical**  
- **Debuggable without reproducing locally**  
- **Consistent across all services**  

---

## Komponen Observability

Observability terdiri dari tiga komponen utama:

| Komponen | Fungsi |
|----------|--------|
| Logging  | Mencatat event dan error |
| Metrics  | Mengukur performa sistem |
| Tracing  | Melacak alur request |

---

## Logging
Ai departemen menggunakan <Link to="../home/tools/loguru">loguru</Link> sebagai standard tool untuk logging.

### Logging dengan Context
Semua log penting harus memiliki context:
```python
from loguru import logger

logger.bind(
    request_id="123",
    service="inference",
    model_version="v1"
).info("Inference started")
```
### Output Behaviour

| Environment | Format        |
| ----------- | ------------- |
| Development | Readable text |
| Production  | JSON          |

```python

```
## Metrics
Metrics digunakan untuk memonitor performa sistem.

Contoh Metrics: 
- Request count
- Latency
- Error rate
- Model accuracy (AI-specific)

## Tracing

Tracing digunakan untuk melacak alur request antar service.

Use Case: 
- Debug latency
- Mengetahui bottleneck
- Memahami flow kompleks
