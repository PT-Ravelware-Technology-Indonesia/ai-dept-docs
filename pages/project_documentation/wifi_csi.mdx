## Notebook 1: CSI-to-Temperature CNN

### Tujuan Riset

CSI amplitude bisa terpengaruh sama **thermal noise** — makin tinggi suhu, makin turun amplitude sinyal secara rata-rata. Notebook ini nyoba liat apakah **CNN bisa belajar pola perubahan amplitude** ini buat memprediksi suhu ruangan.

### Dataset

Struktur file log CSI per baris:

```
CSI_DATA,<mac>,<rssi>,<channel>,...,<timestamp>,<temperature>,[<imag1> <real1> <imag2> <real2> ...]
```

- Setiap baris = satu packet WiFi
- **192 subcarrier** → masing-masing punya pasangan (imag, real)
- **Temperature** = label di kolom ke-24
- Amplitude tiap subcarrier: `|H| = sqrt(real² + imag²)`

Dummy dataset di-generate dengan variasi suhu: **20°C, 22°C, 25°C, 28°C, 32°C** (train) dan **21°C, 30°C** (test — unseen).

untuk percobaan real dapat dilakukan pengumpulan dataset dengan struktur yang sama.

### Sliding Window

Setiap **50 packet berurutan** di-slide jadi satu sample input CNN. Targetnya adalah suhu di packet ke-51. Ini penting soalnya:

- Ngasih konteks temporal ke CNN
- **Nggak nyampurin** antar session — window boundaries nggak pernah lintas file

```
Input shape → (samples, 192 subcarriers, 50 time_steps, 1 channel)
```

### Training

- Optimizer: **Adam** (configurable LR)
- Loss: **MSE**
- Metric: **MAE** (ditampilkan dalam °C real)
- Validation: 10% split dari training data
- Output: loss curves + MAE dalam °C

:::tip

Hasil test MAE dari dummy dataset: **~1.8°C**. Performa di data real mungkin beda — ini cuma bukti konsep.

:::

## Notebook 2: Gesture Recognition (Widar 3.0)

### Tujuan

Notebook ini adalah **eksplorasi awal** buat ngerti struktur data CSI. Pake dataset **Widar 3.0** yang udah publik, isinya **Doppler Spectrum** dari 6 gerakan tangan.

### Dataset Widar 3.0

Data berupa file `.mat` dengan key `doppler_spectrum`:

| Dimensi | Arti |
| ------- | ---- |
| 6 | Rx WiFi receiver ID (antenna) |
| 121 | Doppler frequency bins |
| T | Time steps (variable per session) |

Sumber: [Widar 3.0](http://tns.thss.tsinghua.edu.cn/widar3.0/) — Tsinghua University.

### Preprocessing

- **Downsample** 10× buat ngurangin temporal resolution
- **Normalization** per channel: `(x - min) / (max - min)`
- **Zero-padding**: semua session di-pad ke `T_MAX`
- Final shape: `(N, T_MAX, 6, 121, 1)`

### Model

Arsitektur pake **GRU** (Gated Recurrent Unit):

```text
Input (T_MAX, 6, 121, 1)
  → Reshape/TimeDistributed
  → Bidirectional GRU (64 units)
  → Dense + Dropout
  → Output: 6 classes (gestures)
```

6 gestures yang diklasifikasi: **Push & Pull, Sweep, Clap, Slide, Draw Circle, Draw Cross** (label 1–6).
